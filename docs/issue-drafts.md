# Issue Drafts

## Issue 1

Title: `Runway score should distinguish treasury composition from headline treasury size`

Body:
Right now treasury runway assumes all treasury assets are equally usable. That is too generous. Locked tokens and thin governance allocations are not the same as stable runway capital. We should haircut illiquid treasury components.

## Issue 2

Title: `Unlock overhang needs recipient-aware weighting`

Body:
A 12% unlock to strategic backers is not the same as a 12% unlock to ecosystem rewards or market makers. The current model only prices magnitude. We should also price who receives the supply and how likely it is to hit the market.

Backlog note: tie both issues to actual post-unlock price paths before changing the default model weights.
